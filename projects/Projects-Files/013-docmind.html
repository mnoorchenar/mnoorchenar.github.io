Architecture Overview

LangGraph Cyclic State Machine

DocMind is built around a LangGraph StateGraph with a clean, linear 5-node pipeline. Each node is a specialized agent with a distinct role — only 3 of the 5 make LLM calls, keeping cost and latency low. The Flask backend runs the graph asynchronously via threading, and the frontend polls for trace updates every 1.5 seconds, giving a live view of every agent step.

Score-Based Grading — Fast, Deterministic, Zero API Cost

The Grader doesn't call the LLM at all. It combines the hybrid search score (FAISS + BM25 relevance) with a keyword overlap between the query and each chunk to produce a 0–1 relevance grade instantly. This reduces the pipeline from 8 LLM calls to 3, cutting average query time by ~60%.

Why Use OpenAI SDK Against HuggingFace Router?

The openai Python package is pointed at router.huggingface.co/v1 — the HuggingFace inference router exposes an OpenAI-compatible API. This means the entire LLM layer can be swapped to any other provider (OpenAI, Groq, Ollama) by changing one URL and API key, with zero code changes elsewhere.

Design Decisions

Key Engineering Choices

⚡
3 LLM Calls, Not 8
Replacing the LLM-based grader with a score formula (hybrid score × 0.7 + keyword overlap × 0.3) eliminated 5 sequential LLM calls, cutting average query time by ~60%. The grader is now instant, deterministic, and costs nothing — while answer quality stays the same.
🏠
Local Embeddings = No Rate Limits
Running BAAI/bge-small-en-v1.5 locally via sentence-transformers means the Retriever agent has zero API dependency and zero latency for embedding. Only the 3 LLM reasoning steps hit the free HF API, keeping the system responsive even under multiple concurrent queries.
🔌
Provider-Agnostic LLM Layer
Using the OpenAI Python SDK pointed at router.huggingface.co/v1 means swapping the LLM backend (to OpenAI, Groq, Anthropic, Ollama) requires changing one URL and one API key. The entire agent layer is provider-agnostic by design.
